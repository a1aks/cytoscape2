Command Line Arguments

Cytoscape recognizes a number of optional command line arguments, including run-time specification of network files and expression data:

-g <GML network filename> (xxx.gml) Loads a network file in GML format (see 5. Building and Storing Interaction Networks)

-i <SIF interactions filename> (yyy.sif) Loads a network file in SIF format (see 5. Building and Storing Interaction Networks)

-b <bioData directory> (e.g. annotation/manifest) Specifies which directory to use for the BioDataServer annotations

-e <expression filename> (zzz.pvals) Loads an expression data file (see 6. Loading Gene Expression Data)

-x Prevents expression file from also loading into Cytoscape graph attributes (See 7. Node and Edge Attributes)

-n <nodeAttributes filename> ... (one or more) Loads node attributes files (see 7. Node and Edge Attributes)

-j <edgeAttributes filename> ... (one or more) Loads edge attributes files (see 7. Node and Edge Attributes)

-s< Set the default species name

-c Turn off default node name canonicalization

-t View mode for Cytoscape main window. Can be “tabbed” (default), “internal” or “external”

-h help: display these command line arguments

-v display version

--JLD <directory> specifies a directory in which PlugIn .jar’s reside

--JLW <url>  specifies a URL that points to a PlugIn .jar

--JLL <url> ... specifies a list of URLs pointing to PlugIn .jars

Most data sets may also be loaded after Cytoscape is running. See the sections on 6. Loading Gene Expression Data and 7. Node and Edge Attributes for details.

Additional command line arguments that are not recognized by the Cytoscape core are passed to the PlugIn modules. Please refer to the documentation for each specific PlugIn for more details.
